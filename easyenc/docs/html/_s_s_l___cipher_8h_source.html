33 /*
34  Implements Cipher interface for OpenSSL's ciphers.
35 
36  Design:
37  Variable algorithm, key size, and block size.
38 
39  Partial blocks, keys, and names are encrypted using the cipher in a pseudo
40  stream mode (CFB).
41 
42  Keys are encrypted with 2-4 (KEY_CHECKSUM_BYTES define) checksum bytes
43  derived from an HMAC over both they key data and the initial value vector
44  associated with the key. This allows a good chance at detecting an
45  incorrect password when we try and decrypt the master key.
46 
47  File names are encrypted in the same way, with 2 checksum bytes derived
48  from an HMAC over the filename. This is done not to allow checking the
49  results, but to make the output much more random. Changing one letter in a
50  filename should result in a completely different encrypted filename, to
51  help frustrate any attempt to guess information about files from their
52  encrypted names.
53 
54  Stream encryption involves two encryption passes over the data, implemented
55  as:
56  1. shuffle
57  2. encrypt
58  3. reverse
59  4. shuffle
60  5. encrypt
61  The reason for the shuffle and reverse steps (and the second encrypt pass)
62  is to try and propogate any changed bits to a larger set. If only a single
63  pass was made with the stream cipher in CFB mode, then a change to one byte
64  may only affect one byte of output, allowing some XOR attacks.
65 
66  The shuffle/encrypt is used as above in filename encryption as well,
67  although it is not necessary as they have checksum bytes which augment the
68  initial value vector to randomize the output. But it makes the code
69  simpler to reuse the encryption algorithm as is.
70 */
